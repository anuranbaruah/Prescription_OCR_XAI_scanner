# Fine-tuning TrOCR

The pretrained `microsoft/trocr-base-handwritten` baseline measured by the eval
harness is weak on handwritten medicine names (~10-13% exact word accuracy,
59-70% CER). Fine-tuning on the two bundled datasets is the project's core OCR
contribution.

This folder now has two training paths:

1. `finetune_trocr_colab.ipynb` for longer GPU runs on Colab/Kaggle
2. `train_trocr_local.py` for reproducible local fine-tuning from the repo

## Local training

The local script uses the real bundled handwriting datasets directly from
`<project>/dataset` and writes checkpoints plus metrics under
`backend/results/trocr-rx-finetuned/` by default.

Quick sanity run:

```bat
cd backend
.venv\Scripts\python training\train_trocr_local.py ^
  --epochs 1 ^
  --train-limit 64 ^
  --val-limit 32 ^
  --test-limit 32
```

Longer local run on the full datasets:

```bat
cd backend
.venv\Scripts\python training\train_trocr_local.py ^
  --dataset both ^
  --epochs 6 ^
  --batch-size 1 ^
  --grad-accum 16 ^
  --output-dir results\trocr-rx-finetuned
```

Notes:

- The local environment already has the TrOCR base checkpoint cached, so the
  script defaults to `--local-files-only`.
- On the GTX 1650 in this workspace, use small batches. `--batch-size 1` is the
  safe default; gradient accumulation keeps the effective batch larger.
- `--gradient-checkpointing` is enabled by default to reduce VRAM pressure.
- Validation uses 10% of the `Handwritten Rx` train split plus the bundled BD
  validation split.

To evaluate an existing checkpoint without retraining:

```bat
cd backend
.venv\Scripts\python training\train_trocr_local.py ^
  --eval-only results\trocr-rx-finetuned\best ^
  --output-dir results\trocr-rx-finetuned\eval
```

## Re-measure for the paper

After training finishes, point the eval harness at the fine-tuned checkpoint.

For a single OCR dataset:

```bat
cd backend
set RXAI_TROCR_MODEL=%CD%\results\trocr-rx-finetuned\best
set RXAI_EVAL_OCR_DIR=..\dataset\Handwritten Rx
python -m app.eval.run_ocr_eval
```

For the benchmark JSON used by the frontend:

```bat
cd backend
set RXAI_TROCR_MODEL=%CD%\results\trocr-rx-finetuned\best
python -m app.eval.run_all --ocr-dir "..\dataset\Handwritten Rx" --ocr-dir "..\dataset\Doctor's Handwritten Prescription BD dataset\Testing"
```
